# Brand Style Guide

## Brand personality

- Modern.
- Minimal.
- Athletic.
- Credible.
- Calm but energetic.
- Native/mobile-first.
- Data-rich without feeling stressful.

## Design direction

Statman should feel like a premium sports media app with athlete profiles and stats underneath. It should not feel like a SaaS dashboard, spreadsheet tool, or generic social network.

## Visual principles

- Use the full screen for one meaningful object at a time.
- Prefer white space and strong hierarchy.
- Use media as the emotional layer.
- Use large stat numbers as the credibility layer.
- Use motion for depth and navigation, not decoration.
- Hide complexity until the user asks for it.

## Color strategy

- Base: clean white/off-white backgrounds.
- Text: high-contrast neutral black/charcoal.
- Secondary text: muted neutral gray.
- Accent: one primary brand accent.
- Team colors: allowed as contextual accents, not full theme takeover.
- Status colors: verification, warning, dispute, live, offline.

## Typography

- Entity names: large, confident, bold.
- Stat values: large numerals.
- Labels: compact, uppercase or small muted labels.
- Body: plain, readable, low-friction.

## Spacing

- Large vertical breathing room on profile and feed cards.
- Compact spacing inside stat chips.
- Strong separation between modules.
- Avoid cramped tables on mobile.

## UI language

Preferred:

- Full-bleed media.
- Sliding sheets.
- Sticky mini headers.
- Large cards.
- Stat chips.
- Horizontal rails.
- Native action sheets.
- Swipeable media and entity navigation.

Avoid:

- Dense dashboards.
- Generic Bootstrap cards.
- Table-first pages.
- Heavy borders.
- Overdecorated gradients.
- Tiny touch targets.
