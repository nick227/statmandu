# Accessibility Checklist

## Target standard

WCAG 2.2 AA as the practical target for product design and implementation.

## Global requirements

- All interactive controls must be keyboard accessible on web.
- Touch targets should be large enough for mobile use.
- Text contrast must pass WCAG AA.
- Status should not rely on color alone.
- Motion should respect reduced-motion settings.
- Media must not autoplay with sound.
- Image/video controls must have accessible labels.

## Entity profiles

- Media carousel must expose current item and controls.
- Sliding sheet must be navigable without gestures.
- Stat chips need readable labels and values.
- Verification/dispute badges need text labels, not just icons.

## Live game mode

- Scorekeeper controls need clear labels.
- Undo must be accessible.
- Offline/sync status must be announced visibly and semantically.
- Live score updates should not overwhelm screen readers.
- Spectator timeline should have controlled update behavior.

## Forms

- Inputs require labels.
- Error messages must be specific.
- Required fields must be clear.
- Validation should not trap users.

## Media

- YouTube embeds need titles.
- Player/team images need alt text or decorative handling.
- Fallback media cards should preserve context.
