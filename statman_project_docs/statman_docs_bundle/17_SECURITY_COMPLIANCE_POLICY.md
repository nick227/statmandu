# Security & Compliance Policy

## Data handling principles

- Collect only what is necessary.
- Avoid sensitive data unless required for explicit product features.
- Keep public/private field boundaries clear.
- Treat minors carefully by minimizing unnecessary personal data.
- Allow reporting of impersonation, abuse, and incorrect public info.

## Public profile data

Potentially public:

- Name.
- Sport/team/school/league.
- Position/class year.
- Stats.
- Public media links.
- Recruiting/contact info when user intentionally publishes it.

Potentially private:

- Account email.
- Login credentials.
- Internal verification details.
- Admin notes.
- Device/session info.

## Authentication

- Auth required for edits, follows, reactions, disputes, live scoring roles, team management.
- Admin actions require elevated roles.
- Team managers can approve/add/edit team members depending on team settings.

## Abuse and safety

- Report profile.
- Report media.
- Report stat abuse.
- Report impersonation.
- Block/mute later.
- Comments limited to feed surfaces at first.

## Compliance posture

- Prepare for GDPR/CCPA-style delete/export requests.
- Keep audit logs for meaningful edits/disputes.
- Use source tracking for imported data.
- Avoid direct video hosting in MVP to reduce media moderation burden.
