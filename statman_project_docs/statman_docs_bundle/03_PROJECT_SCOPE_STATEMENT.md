# Project Scope Statement

## Project name

Statman

## In scope for MVP demo

- React Native Web/PWA client.
- OpenAPI-first backend contract.
- Generated SDK foundations.
- Basketball sport config.
- Entity profile shell.
- Player, team, game, league pages.
- Home feed for fresh content.
- Explore surface for rankings/search.
- Live game capture mode.
- Spectator mode.
- Basic broadcaster notes/moment markers.
- End-of-game reconciliation.
- Disputed stat footnotes.
- YouTube media link handling.
- Follow and reaction primitives.
- Source and verification badges.
- Public correction/dispute submission.

## Out of scope for MVP demo

- App store native deployment.
- Direct video upload/hosting.
- Paid billing.
- Full recruiting marketplace.
- Full comment system on profiles.
- Complex AI moderation.
- Every sport’s final stat logic.
- Production-grade third-party data provider integrations.
- Historical source snapshot archival.

## Constraints

- Must be sport-agnostic by architecture.
- First demo must feel highly custom for basketball.
- Must stay mobile-first.
- Must keep cognitive load low.
- Must avoid giant monolithic OpenAPI files.
- Must support future realtime hydration and two-way UI.

## Deliverables

- Public website/PWA.
- Backend API.
- SDK package.
- Sport config package.
- Realtime event contract.
- Demo data set.
- Design system primitives.
- Documentation bundle.

## Acceptance criteria

- A public user can view athletes, teams, games, league, feed, and rankings.
- An athlete profile looks media-rich and credible on mobile.
- A scorekeeper can capture a basketball game quickly.
- Multiple reporter entries can be compared after a game.
- A game can publish with disputed stat footnotes.
- A stat update generates homepage/feed/profile content.
